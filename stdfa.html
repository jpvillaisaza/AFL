Introduction

The aim of this programming lab is to simulate a two-way deterministic finite automaton using Haskell.

Assignment (80%)

  • Download the GitHub repository jpvillaisaza/Automata.
    • The module Automata.Data.TDFA implements two-way deterministic finite automata.
    • The module Automata.Data.TDFA.Examples gives an example of the data type TDFA.
  • (70%) Using the module Automata.Data.TDFA, define a function 
              accepts :: TDFA state symbol -> [symbol] -> Bool
    that tests whether a two-way deterministic finite automaton accepts or rejects an input word.
    • Using the module Automata.Data.TDFA.Examples, for example, accepts m [A, A] = False.
  • (10%) Annotate your source code using Haddock. Documentation should explain your solution for the assignment.

Clean code (10%)

Before submitting your code, clean it up! Clean code:
  • Does not have long lines (at most 80 columns).
  • Has a consistent layout.
  • Has type signatures for everything.
  • Has good comments.
  • Has no junk (unused code, commented code, unnecessary code).
  • Has no overly complicated function definitions.
  • Does not contain any repetitive code.

Submission (10%)

Submission is electronic. Send an email to asicard(at)eafit(dot)edu(dot)co and jvillais(at)eafit(dot)edu(dot)co and attach a compressed file (.zip or .tar.gz) with your solution.

Your submission has to include the following:

  • SimTDFA.hs, a Haskell module file with your solution for the assignment.
  • README (or README.md), a file containing at least the following information.
    • The name of your program.
    • Your name(s).
    • A general description of your program.
    • Information on how to use your program.
